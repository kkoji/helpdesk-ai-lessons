5原則 早見表

原則 一言で 効果
① 具体的に指示する 対象・読み手・分量を明示する 的の絞れた答えになる
② 役割を与える systemで「あなたは〜です」と伝える 口調・視点・守備範囲が安定する
③ 出力形式を指定する 箇条書き・表・JSONを指定する そのまま使える／コードで処理できる
④ 例を見せる お手本を1〜2個添える 型を真似てくれる
⑤ 「わからないとき」を決める 逃げ道を用意する ハルシネーション対策の第一歩になる
💡5つとも、AIは前提を共有していない相手なので、人間相手なら省略できることまで言葉にして渡す、という一点から導かれています。迷ったらこの原点に戻ってください。

原則別テンプレート集

{ } の部分（コード内で { 黄色 } で表示）を、ご自身の内容に書き換えて使ってください。

1具体的に指示する

指定しなかった条件は、AIが勝手に決めます。決めてほしくないことは、こちらが先に決めて渡します。

テンプレート 
{読み手}向けに、{対象}について、{分量・形式}で説明してください。
特に{必ず触れてほしい点}に触れてください。
記入例 
新入社員向けに、交通費の経費精算の手順を、3ステップで説明してください。
特に提出期限に必ず触れてください。
2役割を与える

役割を決めないと、AIは「誰として答えるか」を毎回自分で選びます。立場を固定すれば、答えのぶれが小さくなります。systemメッセージに書きます。

テンプレート 
あなたは{役割}です。
{口調・スタイルの指示}で答えてください。
{守備範囲の制約}。
記入例 
あなたは社内ヘルプデスクの担当者です。
丁寧な日本語で、簡潔に答えてください。
社内の問い合わせにのみ答え、それ以外の話題には「お答えできません」と回答してください。
3出力形式を指定する

答えの形を先に決めると、そのまま使える出力になります。プログラムで後処理する場合は特に重要です。

テンプレート 
次の形式で答えてください。
【結論】（1行）
【手順】（番号付きの箇条書き、最大5項目）
【問い合わせ先】（部署名）

プログラムで後処理する場合（JSON指定）

テンプレート 
必ず次のキーを持つJSONのみを出力してください。説明文は不要です。
{"answer": "回答本文", "department": "担当部署名"}
⚠️JSON形式を指定しても、まれに余計な文が混ざることがあります。アプリに組み込むときは、受け取り側での検証も忘れずに行ってください（コース後半で扱います）。
4例を見せる（Few-shot）

文章で細かく形式を指定しようとすると、指示自体が長く複雑になります。お手本を1つ見せるほうが、短くて誤解も生みません。

テンプレート 
以下の回答例と同じ型で答えてください。

# 回答例
質問: 社用携帯を紛失しました
【結論】直ちに利用停止の手続きが必要です
【手順】1. 情報システム部に電話 2. 紛失届を提出
【問い合わせ先】情報システム部

質問: {question}
5「わからないとき」の振る舞いを決める

AIに「わからないと言っていい」と許可を与えます。ハルシネーション対策の第一歩です。

テンプレート 
確かな情報がない場合は、推測で答えず、正直に
「わかりません。{案内先}にお問い合わせください」と答えてください。
記入例 
確かな情報がない場合は、推測で答えず、正直に
「わかりません。総務部（内線1234）にお問い合わせください」と答えてください。
🔑この一文は、このコースで作る社内ヘルプデスクAIのシステムプロンプトにも入る、最重要の型です。ただしプロンプトだけでは完全には防げないため、この後に学ぶ RAG と組み合わせます。

LangChainで使える完成形

テンプレートを ChatPromptTemplate に落とし込んだ、そのまま実行できるコードです。

基本形：ヘルプデスクQ&A（原則①②⑤入り）

Python 
from langchain_openai import ChatOpenAI
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.output_parsers import StrOutputParser

prompt = ChatPromptTemplate.from_messages([
    ("system",
     "あなたは社内ヘルプデスクの担当者です。"
     "丁寧な日本語で、簡潔に答えてください。"
     "確かな情報がない場合は、推測で答えず、正直に"
     "「わかりません。担当部署にお問い合わせください」と答えてください。"),
    ("user", "{question}"),
])
llm = ChatOpenAI(model="gpt-4o-mini")
parser = StrOutputParser()

chain = prompt | llm | parser
print(chain.invoke({"question": "有給休暇の申請方法を教えてください"}))

応用形：型を固定した回答（原則③④入り）

Python 
prompt = ChatPromptTemplate.from_messages([
    ("system",
     "あなたは社内ヘルプデスクの担当者です。"
     "以下の回答例と同じ型で答えてください。\n\n"
     "# 回答例\n"
     "質問: 社用携帯を紛失しました\n"
     "【結論】直ちに利用停止の手続きが必要です\n"
     "【手順】1. 情報システム部に電話 2. 紛失届を提出\n"
     "【問い合わせ先】情報システム部"),
    ("user", "質問: {question}"),
])

chain = prompt | llm | parser
print(chain.invoke({"question": "会議室の予約方法がわかりません"}))

応用形：スタイルを差し替えられるプレースホルダー2つのテンプレート

Python 
prompt = ChatPromptTemplate.from_messages([
    ("system", "あなたは社内ヘルプデスクの担当者です。{style}で答えてください。"),
    ("user", "{question}"),
])

chain = prompt | llm | parser
print(chain.invoke({"style": "箇条書きで3点以内", "question": "在宅勤務の申請方法は？"}))

書き方に迷ったときのチェックリスト

  • 誰向けの答えか書いたか（読み手）
  • 何についてか絞ったか（対象）
  • どのくらいの分量・形で欲しいか書いたか（分量・形式）
  • systemメッセージで役割と守備範囲を与えたか
  • わからないときの逃げ道を用意したか
  • 型が大事なら、お手本を1つ見せたか

補足

  • テンプレートの文言は日本語のまま使えます（gpt-4o-mini は日本語の指示を問題なく理解します）。
  • プロンプトは「書いたら終わり」ではありません。出力を見て少しずつ直す のが前提です。1回で完璧を狙わず、悪い例と良い例を比べながら調整してください。
  • ここに載せた型は、この後のRAG（検索した文書を差し込む）やエージェント（ツールの使い分け）でも、土台として登場します。
