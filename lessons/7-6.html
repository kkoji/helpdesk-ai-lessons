セクション7 AWSへの展開：実運用化

【ラボ8 前編】FAISSインデックスの構築とS3格納 ｜ 実行コマンド一覧

AWS CloudShell（ブラウザ内のシェル）で実行するコマンドです。

使い方：各ブロック右上の「コピー」ボタンを押して、CloudShellに貼り付けて実行します。複数行を貼り付けると「複数行のテキストの安全な貼り付け」という確認画面が出るので、内容を確認して「貼り付け」を押してください。
環境ごとの進め方の違い
・UdemyのAWSラボ環境の方：ラボ環境はレクチャーごとにリセットされ、前のラボで作ったリソースは残っていません。このページのブロックを上から順にすべて実行してください（「スキップ可」のブロックも実行が必要です）。
・ご自身のAWSアカウントの方：前のラボまでのリソースが残っているため、「スキップ可」のブロックは飛ばせます。ただし変数設定を含むブロック（土台の準備・URL取得）は毎回実行してください（変数はターミナルのセッションを閉じると消えるため）。

1準備：ソースコードの取得・変数の設定・バケットの用意

リポジトリの取得・変数の設定・バケットの用意までをまとめて行います。OPENAI_API_KEY はご自身のキーに書き換えてください。

# ソースコードを取得（取得済みの場合はエラーが表示されますが、そのまま進んでOK）
cd ~
git clone https://github.com/kkoji/helpdesk-ai.git
cd ~/helpdesk-ai

export AWS_REGION=us-east-1
export ACCOUNT_ID=$(aws sts get-caller-identity --query Account --output text)
export S3_BUCKET=helpdesk-ai-${ACCOUNT_ID}-${AWS_REGION}
export OPENAI_API_KEY=sk-...   # 自分のキー

# 作業用のS3バケットを用意（前のラボで作成済みなら既存を使う。ラボを起動し直した場合はここで作成される）
aws s3 mb s3://${S3_BUCKET} --region ${AWS_REGION} 2>/dev/null || echo "既存バケットを使用: ${S3_BUCKET}"

2社内文書8件とメタデータをS3にアップロード

社内文書8件を docs/ へ、metadata.json をバケット直下へアップロードします。

aws s3 cp data/docs/ s3://${S3_BUCKET}/docs/ --recursive
aws s3 cp data/metadata.json s3://${S3_BUCKET}/metadata.json

3依存ライブラリを入れて、FAISSインデックスを構築（Embeddings課金がこの1回だけ発生）

依存ライブラリを入れて、build_index.py でインデックスを構築します。Embeddingsの課金が発生するのはこの1回だけです。

python3 -m pip install -r aws/requirements.txt --quiet
python3 aws/build_index.py \
  --docs-dir data/docs \
  --metadata data/metadata.json \
  --output-dir vector_index

4生成されたインデックスファイルをS3へアップロードし、確認

生成された index.faiss と chunks.json を vector_index/ へアップロードし、4点セットがそろったことを一覧で確認します。

aws s3 cp vector_index/index.faiss s3://${S3_BUCKET}/vector_index/index.faiss
aws s3 cp vector_index/chunks.json s3://${S3_BUCKET}/vector_index/chunks.json

aws s3 ls s3://${S3_BUCKET} --recursive
