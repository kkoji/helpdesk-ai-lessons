セクション7 AWSへの展開：実運用化

コスト管理とクリーンアップ ｜ 実行コマンド一覧

このセクションで作成したAWSリソースの削除コマンドです。AWS CloudShellで実行します。

使い方：各手順の右上にある「コピー」ボタンを押して、CloudShellに貼り付けて実行します。複数行を貼り付けると「複数行のテキストの安全な貼り付け」という確認画面が出るので、内容を確認して「貼り付け」を押してください。
環境ごとの違い
・UdemyのAWSラボ環境の方：ラボ環境はセッション終了時にリセットされるため、基本的に手動での削除は不要です。
・ご自身のAWSアカウントの方：リソースを残すと課金が続くため、確認が済んだら必ず削除を実施してください。

1リソースを個別に削除する場合

ラボで手作りしたリソースをまとめて削除します。⑦のバケット削除は中身ごと消え、取り消せません。

# 変数（ラボと同じもの）
export AWS_REGION=us-east-1
export ACCOUNT_ID=$(aws sts get-caller-identity --query Account --output text)
export S3_BUCKET=helpdesk-ai-${ACCOUNT_ID}-${AWS_REGION}

# ① API Gateway（チャットボット公開のラボで作成）
API_ID=$(aws apigatewayv2 get-apis \
  --query "Items[?Name=='helpdesk-lab09-api'].ApiId | [0]" --output text)
[ "$API_ID" != "None" ] && aws apigatewayv2 delete-api --api-id ${API_ID}

# ② Lambda関数（各ラボで作成した3つ）
for FN in helpdesk-lab07 helpdesk-lab08b helpdesk-lab09; do
  aws lambda delete-function --function-name ${FN} 2>/dev/null && echo "削除: ${FN}"
done

# ③ CloudWatch Logs（関数を消しても残るので明示的に削除）
for FN in helpdesk-lab07 helpdesk-lab08b helpdesk-lab09; do
  aws logs delete-log-group --log-group-name /aws/lambda/${FN} 2>/dev/null
done

# ④ Lambda Layer（全バージョンを削除）
for V in $(aws lambda list-layer-versions --layer-name helpdesk-ai-deps \
  --query "LayerVersions[].Version" --output text); do
  aws lambda delete-layer-version --layer-name helpdesk-ai-deps --version-number ${V}
done

# ⑤ DynamoDBテーブル（会話履歴のラボで作成）
aws dynamodb delete-table --table-name helpdesk-sessions 2>/dev/null

# ⑥ IAMロール（インラインポリシー→管理ポリシーの順に外してから削除）
aws iam delete-role-policy --role-name helpdesk-ai-lambda-role \
  --policy-name helpdesk-ai-inline 2>/dev/null
aws iam detach-role-policy --role-name helpdesk-ai-lambda-role \
  --policy-arn arn:aws:iam::aws:policy/service-role/AWSLambdaBasicExecutionRole 2>/dev/null
aws iam delete-role --role-name helpdesk-ai-lambda-role 2>/dev/null

# ⑦ S3バケット（中身ごと削除。取り消せないので注意！）
aws s3 rb s3://${S3_BUCKET} --force

2（参考）destroy.shで一括削除する場合参考

deploy.sh で一括構築した環境は、付属の destroy.sh で一括削除できます。どちらで作ったか不明な場合は両方実行しても安全です。

cd helpdesk-ai/aws
./destroy.sh          # 削除対象を表示し、確認プロンプト（yes入力）の後に削除
FORCE=1 ./destroy.sh  # 確認なしで即削除

3削除できたか確認

それぞれ「空」または「見つからない」と表示されればクリーンアップ完了です。

aws lambda list-functions --query "Functions[?starts_with(FunctionName, 'helpdesk')].FunctionName"
aws apigatewayv2 get-apis --query "Items[?starts_with(Name, 'helpdesk')].Name"
aws dynamodb list-tables --query "TableNames[?starts_with(@, 'helpdesk')]"
aws s3 ls | grep helpdesk || echo "S3: なし"
aws iam get-role --role-name helpdesk-ai-lambda-role 2>&1 | head -1
