② 本番化チェックリスト

セキュリティの回でまとめた本番化の要点です。ご自身のAIを社内に出す前に、ひととおり確認してください。

  • 秘密情報：キーはコード・Gitに無い／環境変数 or Secrets Manager／不要キーはRevoke済み
  • 最小権限：IAMポリシーに * が無い／サービスごとに必要な操作だけ／ルートユーザーを日常使いしない
  • 公開範囲：CORSを自社ドメインに限定／APIに認証（簡易トークン／IAM認証／Cognito等）
  • LLMリスク：「読ませる＝送る」の確認（機密区分・提供元のデータポリシー）／プロンプトインジェクションへの備え（防御を過信しない・ツール権限を絞る・出力を無条件に信用しない）
  • コスト：AWS Budgetsのアラート設定／OpenAIの自動チャージ設定確認／不要リソースの削除（コスト管理とクリーンアップの回を参照）
  • 運用：テスト質問セットで回帰確認する習慣／CloudWatch Logsでエラーを確認できる

③ 発展トピックの学習キーワード集

検索エンジンや公式ドキュメントで調べるときの「入口の言葉」です。

Advanced RAG（検索を極める）

  • Re-ranking / リランキング（二段選抜）
  • ハイブリッド検索 / hybrid search・BM25（キーワード検索の代表的手法）
  • クエリ書き換え / query rewriting・HyDE
  • チャンク戦略 / chunking strategy・親子チャンク / parent document retriever

エージェント（チームにする）

  • LangGraph multi-agent
  • ReAct（推論と行動を繰り返すエージェントの基本パターン。本コースの「判断→行動→観察」の原型）
  • Function Calling / Tool Use
  • Human-in-the-loop（人間の承認を挟む設計。エスカレーションの発展形）

評価（測り続ける）

  • RAG evaluation / RAG 評価
  • LLM-as-a-Judge（LLMに採点させる）
  • 忠実性 / faithfulness・関連性 / relevancy（RAG評価の代表的観点）

運用・その他

  • プロンプトインジェクション対策
  • LLM オブザーバビリティ / observability（トレース・ログの可視化）
  • セマンティックキャッシュ（同じ質問への回答を再利用）

④ 公式ドキュメントの歩き方

調べたいこと行き先
LangChain／LangGraph のAPI・使い方LangChain公式ドキュメント（python.langchain.com）・LangGraphドキュメント
OpenAI APIの仕様・モデル・料金OpenAIプラットフォームのドキュメントと料金ページ（platform.openai.com）
Lambda／API Gateway／S3／DynamoDBAWS公式ドキュメント。まず「開発者ガイド」の入門章から
FAISSFAISSのGitHubリポジトリのwiki

歩き方のコツ

  1. バージョンに注意：LangChain系は進化が速く、古い記事のコードは動かないことがあります。公式ドキュメント→リリースノートの順で確認を。
  2. 「動く最小コード」を先に確保：本コースのリポジトリ（helpdesk-ai）は動作する全体像として機能します。新技術は、まずこの上に小さく足して試すのが安全です。
  3. エラー文で検索：エラーメッセージの固有部分（クラス名・エラーコード）をそのまま検索するのが最短路です。
