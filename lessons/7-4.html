セクション7 AWSへの展開：実運用化

【ラボ7】LambdaでOpenAI APIを呼び出す ｜ 実行コマンド一覧

AWS CloudShell（ブラウザ内のシェル）で実行するコマンドです。

使い方：各ブロック右上の「コピー」ボタンを押して、CloudShellに貼り付けて実行します。複数行を貼り付けると「複数行のテキストの安全な貼り付け」という確認画面が出るので、内容を確認して「貼り付け」を押してください。
環境ごとの違い
・UdemyのAWSラボ環境の方：ラボ環境はレクチャーごとにリセットされるため、このラボで作ったリソースは次のラボには残りません。各ラボの冒頭で同じ土台を作り直せるようにしてあるので、心配はいりません。
・ご自身のAWSアカウントの方：作ったリソースはそのまま残るため、削除しない限り、以後のラボで同じリソースを作り直す必要はありません。

1このラボで使う変数をまとめて設定（OPENAI_API_KEY は自分のキーに書き換える）

このラボで使う値を環境変数にまとめて設定します。最後にAPIキーの入力待ちになるので、ご自身のキーを貼り付けて Enter を押してください（画面には表示されません）。

export AWS_REGION=us-east-1
export ACCOUNT_ID=$(aws sts get-caller-identity --query Account --output text)
export S3_BUCKET=helpdesk-ai-${ACCOUNT_ID}-${AWS_REGION}
export LAMBDA_ROLE_NAME=helpdesk-ai-lambda-role
export LAYER_NAME=helpdesk-ai-deps
echo ${S3_BUCKET}

# OpenAI APIキーの設定（入力待ちになったらキーを貼り付けて Enter。画面には表示されない。設定済みならそのまま使われる）
until [ -n "${OPENAI_API_KEY}" ]; do read -s -p "OpenAI APIキーを貼り付けて Enter: " OPENAI_API_KEY; echo ""; done; export OPENAI_API_KEY; echo "APIキーを設定しました（${#OPENAI_API_KEY} 文字）"

2作業用のS3バケットを作成

作業用のS3バケットを作成します。まずはLayer zipの置き場として使い、この先のラボでは文書やインデックスの置き場にもなります。

aws s3 mb s3://${S3_BUCKET} --region ${AWS_REGION}

3Lambdaの実行ロールを作成

Lambdaに権限を与える実行ロールを作成し、ログ書き込みの基本権限を付けます。

aws iam create-role \
  --role-name ${LAMBDA_ROLE_NAME} \
  --assume-role-policy-document '{
    "Version": "2012-10-17",
    "Statement": [{
      "Effect": "Allow",
      "Principal": {"Service": "lambda.amazonaws.com"},
      "Action": "sts:AssumeRole"
    }]
  }'

aws iam attach-role-policy \
  --role-name ${LAMBDA_ROLE_NAME} \
  --policy-arn arn:aws:iam::aws:policy/service-role/AWSLambdaBasicExecutionRole

export LAMBDA_ROLE_ARN=arn:aws:iam::${ACCOUNT_ID}:role/${LAMBDA_ROLE_NAME}

4Lambda Layer（依存ライブラリ）をS3経由で登録

依存ライブラリ入りのLayer zipをコース配布のバケットから取得し、自分のバケット経由でLayerとして登録します（直接アップロードの上限を超えるためS3経由にしています）。

aws s3 cp s3://udemy-helpdesk-public/layers/layer-v1.zip layer.zip --no-sign-request

aws s3 cp layer.zip s3://${S3_BUCKET}/layers/layer-v1.zip

export LAYER_VERSION_ARN=$(aws lambda publish-layer-version \
  --layer-name ${LAYER_NAME} \
  --content S3Bucket=${S3_BUCKET},S3Key=layers/layer-v1.zip \
  --compatible-runtimes python3.11 \
  --query LayerVersionArn --output text)

echo ${LAYER_VERSION_ARN}

5Lambda関数を作成

Lambda関数のコードを取得してzipに固め、関数を作成します。APIキーとモデル名は環境変数で渡します。

aws s3 cp s3://udemy-helpdesk-public/src/lab07_lambda_function.py . --no-sign-request

zip function.zip lab07_lambda_function.py

aws lambda create-function \
  --function-name helpdesk-lab07 \
  --runtime python3.11 \
  --role ${LAMBDA_ROLE_ARN} \
  --handler lab07_lambda_function.lambda_handler \
  --zip-file fileb://function.zip \
  --layers ${LAYER_VERSION_ARN} \
  --timeout 60 --memory-size 512 \
  --environment "Variables={OPENAI_API_KEY=${OPENAI_API_KEY},CHAT_MODEL_ID=gpt-4o-mini}"

6クラウドのLambdaに質問を送る

クラウドで動いているLambdaに質問を送り、応答が返ることを確認します。

aws lambda invoke \
  --function-name helpdesk-lab07 \
  --cli-binary-format raw-in-base64-out \
  --payload '{"message": "こんにちは！あなたはどこで動いていますか？"}' \
  response.json

jq -r '.reply' response.json
