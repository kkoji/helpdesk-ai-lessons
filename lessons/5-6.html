セクション5 RAGチャットボットの完成

複数形式のドキュメント対応（PDF等）

PDF・Word・CSVからテキストを取り出して、コースで作ったRAGパイプラインに流し込む方法

📂
このコースの社内文書サンプルはテキストファイル（Markdown形式）ですが、実際の職場の文書は PDF・Word・CSV など様々です。それらに対応するための、コピペで使えるコード例集です。
ブックマークしておいてください。自分の会社の文書でRAGを試すときに、このページに戻ってきてください。

基本の考え方：入口だけ差し替える

コースで作ったRAGの流れはこうでした。

文書ファイル → テキスト → チャンク分割 → ベクトル化 → FAISS

青い部分（ファイル→テキスト）だけを各形式用に差し替えます

PDF等に対応するのに、チャンク分割から先を作り直す必要はありません。「ファイル→テキスト」の入口部分だけ を各形式用のローダーに差し替えれば、あとは同じパイプラインがそのまま使えます。

LangChainには各形式用の Document Loader が揃っています。多くは langchain-community パッケージに含まれるため、まず以下をインストールしてください。

ターミナル 
pip install langchain-community
💡各ローダーが返すのは Document オブジェクトのリストです。本文は .page_content、ファイル名やページ番号などは .metadata に入っています。

形式別コード例

📕PDF（PyPDFLoader）

社内規程・マニュアルで最も多い形式。1ページが1つの Document になります。

ターミナル 
pip install pypdf
Python 
from langchain_community.document_loaders import PyPDFLoader

loader = PyPDFLoader("就業規則.pdf")
pages = loader.load()          # 1ページ = 1 Document のリスト

print("ページ数:", len(pages))
print(pages[0].page_content[:200])   # 1ページ目の本文
print(pages[0].metadata)             # {'source': '就業規則.pdf', 'page': 0, ...}
⚠️PDFは見た目重視の形式のため、レイアウトによっては文字の順番が崩れることがあります（段組み・表・ヘッダーフッターなど）。取り出したテキストを必ず目視確認してください。
⚠️スキャン画像のPDF（紙をスキャンしたもの）は文字データを持っていないため、この方法では読めません。OCR（光学文字認識）という別の処理が必要です（本コースの範囲外）。
📘Word（Docx2txtLoader）

通常はファイル全体で1つの Document になります。

ターミナル 
pip install docx2txt
Python 
from langchain_community.document_loaders import Docx2txtLoader

loader = Docx2txtLoader("経費精算マニュアル.docx")
docs = loader.load()           # 通常はファイル全体で 1 Document

print(docs[0].page_content[:200])
⚠️対応しているのは .docx（新形式）です。古い .doc は事前にWordで .docx に保存し直してください。
📗CSV（CSVLoader）

1行が1つの Document になります。追加インストールは不要です。

Python 
from langchain_community.document_loaders import CSVLoader

loader = CSVLoader("faq一覧.csv", encoding="utf-8")
rows = loader.load()           # 1行 = 1 Document

print(rows[0].page_content)    # 例: "質問: ○○\n回答: △△" のように、各列が「列名: 値」の形式で整形される
🔑FAQのように1行が十分短く、意味的にも独立しているデータなら、チャンク分割を行わずそのままベクトル化して問題ありません。
📙テキスト・Markdown（コースで使った方法）

参考までに、コースの方式です。標準ライブラリのみで、ローダーは不要です。

Python 
from pathlib import Path

text = Path("就業規則.md").read_text(encoding="utf-8")

コースのパイプラインへのつなぎ込み

ローダーで取り出したテキストを、文書の読み込みとチャンク分割のハンズオンで作った関数に流し込む例です（PDFの場合）。

Python 
# ① PDF からテキストを取り出す（全ページを連結）
loader = PyPDFLoader("就業規則.pdf")
pages = loader.load()
doc_text = "\n".join(page.page_content for page in pages)

# ② あとはコースと同じ：チャンク分割 → 出典付きチャンクへ
#    （split_into_sections / split_long / build_chunks_for_doc はハンズオンで作ったもの）
doc_meta = {"title": "就業規則", "department": "人事部"}
chunks = build_chunks_for_doc("就業規則.pdf", doc_text, doc_meta)
print("チャンク数:", len(chunks))

チャンク分割の注意

コースの split_into_sections は ##  見出しを前提にしています。PDFやWordから取り出したテキストには ##  が無いことが多いため、次のどちらかで対応してください。

1. 前処理で見出しを付ける — 取り出したテキストの章タイトル行に ##  を付けて保存し直す（文書数が少なければ確実でおすすめ）

2. split_long だけで切る — 見出し分割を諦め、上限＋オーバーラップの再分割だけを使う（手軽だが、意味の切れ目は考慮されない）

Python 
# 方法2の例：見出しなしで、上限400字＋オーバーラップ50字だけで切る
pieces = split_long(doc_text, MAX_CHUNK_CHARS, OVERLAP_CHARS)

トラブルシューティング

  • 文字化けする
    encoding="utf-8" を指定する。それでも化ける場合は元ファイルの文字コード（Shift_JIS等）を確認。
  • PDFから何も取れない／記号だらけ
    スキャン画像PDFの可能性。OCRが必要（本コース範囲外）。
  • 表の中身がぐちゃぐちゃになる
    PDFの表は苦手分野。重要な表は、手動でテキスト・CSVに書き起こすのが確実。
  • チャンクを見たら意味不明な断片だらけ
    レクチャー「RAGの精度を高めるコツ」の大原則どおり、まず取り出したテキストを目視で確認。入口のテキストが崩れていれば、その先も全部崩れます。
